# Чек-лист перед публикацией кейса

## Правки в макетах

- [ ] «Отправление через 32 мин» → «Прибытие через 32 мин» (экран закреплённого маршрута)
- [ ] Решить по термину «Табельный №»: оставить (слово общеупотребимое, NDA не нарушает) или заменить на «Номер сотрудника» (понятнее). В тексте и на экранах — одинаково
- [ ] Проверить каждый экран: все имена, номера автобусов, адреса, названия остановок — вымышленные

## Правки на странице кейса

- [ ] Опечатки и двойные пробелы (в т.ч. «Предзаказ  создан»)
- [ ] Точки в заголовках блоков — убрать, привести к единому виду («Предварительный заказ.» → «Предварительный заказ»)
- [ ] В блок QR добавить фразу про автообновление кода: «код автоматически обновляется — пересланный скриншот к моменту посадки уже недействителен»
- [ ] Блок «цикл планирования» — дать подзаголовок («Логика продукта») или перенести в конец после решений как вывод
- [ ] Проверить наличие шапки: заголовок, «Внутренний продукт для крупного банка · NDA», роль, платформа, «экраны показаны с вымышленными данными»
- [ ] Финальная строка: «Детали проекта и остальные сценарии могу показать и рассказать на встрече»

## Состав экранов витрины (4–5)

1. Главный экран разового сотрудника (предварительный заказ)
2. Главный экран закреплённого сотрудника (карта, прибытие) — рядом с первым, как пара
3. QR-билет
4. Обращение — шаг выбора поездки (+ форма с фото)

Не включать: профиль, список поддержки, типовые экраны.
