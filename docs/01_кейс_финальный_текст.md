# Сервис заказа корпоративного транспорта

Мобильное приложение · Внутренний продукт для крупного банка · NDA

---

## Контекст

**Роль:** UX/UI-дизайнер
**Платформа:** iOS / Android
**Формат:** продукт под NDA — название и заказчик не раскрываются, экраны показаны с вымышленными данными

## О продукте

Приложение для сотрудников компании: служебный транспорт до работы. Два сценария — персональный предзаказ с выбором направления, даты и времени или закреплённый служебный автобус, который назначается автоматически.

## Продуктовый контекст

Корпоративная развозка решает типовые задачи крупного работодателя: доступность офисов, куда сложно добраться общественным транспортом, предсказуемое начало смен и транспортная льгота как часть соцпакета. Приложение при этом даёт бизнесу то, чего не даёт «автобус по расписанию»: данные о реальном спросе для оптимизации маршрутов и контроль посадки по QR.

## Моя роль

Спроектировала и отрисовала ключевые сценарии: заказ транспорта, QR-билет, история поездок, профиль, обращения в поддержку и оценка качества услуг. Корректировала пользовательские сценарии под бизнес-цели, провела конкурентный анализ.

## Один интерфейс — два типа пользователей

- **Персональный предзаказ:** сотрудник сам выбирает направление и время — интерфейс строится вокруг формы заказа и подтверждённой поездки.
- **Закреплённый автобус:** транспорт назначается организацией, сотруднику не нужно ни одного обязательного действия — интерфейс строится вокруг карты с маршрутом и временем прибытия.

Я проектировала оба сценария как один путь: сначала сотрудник заказывает поездки вручную, потом его закрепляют за маршрутом — и действия исчезают. Экраны в обоих случаях собраны из одних элементов: карта, карточка поездки, QR. При переходе на постоянный маршрут человек ничего не переучивает — интерфейс просто становится проще.

## Решения

### Предзаказ — для сотрудников с нерегулярными поездками111

Сменные и гибридные графики, разовые поездки, работа в нетиповое время — для этих сценариев ежедневный маршрут избыточен. Предзаказ позволяет выбрать направление, дату и время подачи транспорта. Для бизнеса заявки делают нерегулярный спрос измеримым: транспорт назначается по фактической потребности, а не резервируется заранее.

### Закреплённый маршрут — для регулярных поездок

Сотрудники с постоянным графиком закрепляются за маршрутом с регулярными рейсами — транспорт назначается автоматически, ежедневные действия не требуются. Экран отвечает на главный вопрос — через сколько минут автобус прибудет на остановку; положение машины на карте и расписание каждой остановки доступны по касанию. Бизнес получает предсказуемую загрузку маршрутов и планирование по фактическим спискам.

*[Экран: карта с маршрутом, «Прибытие через N мин», данные автобуса, чипсы рейсов]*

### QR-билет — контроль посадки

Водитель сканирует код при входе. Для компании это решает две задачи: транспортом пользуются только сотрудники — код привязан к учётной записи и автоматически обновляется, пересланный скриншот к моменту посадки уже недействителен, — а учёт поездок ведётся сам: кто, когда и каким рейсом, без бумажных списков.

*[Экран: QR-билет с данными сотрудника]*

### Обращения — с контекстом поездки

Вместо свободного текста — структурированные данные. Поездка из истории, тип обращения и фото позволяют диспетчеру быстрее обрабатывать заявки, а бизнесу — анализировать качество маршрутов и работу перевозчиков. Шаг выбора поездки можно пропустить: не каждый вопрос связан с поездкой.

Моя задача здесь была продуктовой: спроектировать сбор обращений так, чтобы на выходе получались данные, а не свободный текст. Выбор поездки из истории вместо текстового описания, типы обращений вместо общей формы, фото как доказательство — каждое обращение приходит диспетчеру готовым к обработке.

## Логика продукта — как сценарии связаны

Цикл устроен так: сначала сотрудник заказывает поездки вручную, потом система видит регулярность — и появляется постоянный маршрут, где заказывать уже ничего не нужно. Чем дольше живёт продукт, тем меньше действий требует.
