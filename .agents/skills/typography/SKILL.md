# Типографика проекта

Применяется ко всем текстам портфолио: макеты в Figma, страницы сайта на GitHub Pages, markdown-документы.

## Висячие предлоги

Короткие предлоги, союзы и частицы не должны оставаться в конце строки — после них ставится неразрывный пробел (U+00A0; в HTML — `&nbsp;`).

Список слов: в, во, и, а, но, не, ни, на, с, со, к, ко, у, о, об, от, до, по, за, из, для, при, под, над, без, как, что, же, бы, ли (и их варианты с заглавной буквы).

Также неразрывный пробел ставится **перед** тире: «слово&nbsp;— слово».

В Figma: заменять пробел после предлога на символ U+00A0 прямо в `characters`. В HTML: `&nbsp;`.

## Регистр: без капса

Текст не набирается сплошными заглавными буквами (КАПСОМ) — ни в подписях, ни в чипсах, ни в служебных строках. Используется обычный регистр предложения. Аббревиатуры (NDA, QR, iOS, UX/UI) сохраняют своё написание.

Для таких подписей используется обычный текстовый шрифт проекта (Onest), а не моноширинный.

## Шрифты проекта (концепт «Маршрут»)

- Заголовки: Unbounded (SemiBold — хиро, Medium — секции)
- Текст и подписи: Onest (Regular — абзацы, Medium — лиды и подписи)
- Моноширинный JetBrains Mono — только для цифр-маркеров (01–04), не для слов
